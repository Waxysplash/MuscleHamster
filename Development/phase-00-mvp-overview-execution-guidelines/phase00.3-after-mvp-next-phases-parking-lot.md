# Phase 00.3 — After MVP: Next Phases (Parking Lot)

## Header
- **Goal:** Capture the next logical feature phases to add once MVP is done satisfactorily.
- **Status:** Planning
- **Priority:** MEDIUM
- **Dependencies:** Phase 00.1
- **Platforms:** iOS (15+) initially

---

## Overview

### Problem / Motivation
We want a clear post-MVP path without allowing post-MVP scope to creep into MVP execution.

### Next Logical Feature Phases
- **Social**
  - Friends (add via username/contacts/link/QR)
  - View friends’ hamsters + progress
  - Friend streaks and restoration payment options
  - Privacy controls: blocking, profile visibility
- **Monetization**
  - Rewarded ads for bonus points
  - Banner/interstitial ads with safe placements and frequency caps
  - Economy tuning and playtesting
- **Recommendations v1+**
  - More adaptive personalization once enough workout history exists
  - Better explainability and user control for suggestions
- **Privacy & trust polish**
  - Full account deletion flow
  - Data export flow
  - Clearer transparency screens for data use
- **Offline-first**
  - Define what works offline and what requires connectivity
  - Reconnection behavior, sync expectations, and conflict strategy

---

## Deliverables
- A post-MVP backlog that can be turned into phases when MVP is signed off.
