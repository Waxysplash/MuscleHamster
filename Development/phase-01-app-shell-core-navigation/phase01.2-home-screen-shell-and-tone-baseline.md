# Phase 01.2 — Home Screen Shell and Tone Baseline

## Header
- **Goal:** Create a non-functional Home screen shell that establishes layout and copy tone for the app.
- **Status:** Implemented
- **Priority:** HIGH
- **Dependencies:** Phase 01.1
- **Platforms:** iOS (15+)

---

## UX & Screens (Mobile-First)

### Home screen content (placeholders)
- Hamster display area (placeholder)
- Streak area (placeholder)
- “Today status” area (placeholder)
- Primary call-to-action area (placeholder for “Start Workout” / “Check in” later)

### Tone requirements (must-haves)
- Copy is warm, nurturing, and hamster-voiced
- No guilt/shame language (no “you missed”, “falling behind”, etc.)

### States
- First-run empty (encouraging, simple next step)
- Generic error (helpful recovery suggestion)

## Deliverables
- Home screen shell that sets the emotional tone and creates a stable space for later features.
